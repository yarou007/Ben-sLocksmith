Start with the problem

Match duplication controls to the property workflow

A restricted-key plan combines compatible cylinders and keys with an authorization process for requesting additional keys. The practical value depends on how the property approves, issues, records and recovers keys.

Restrictions and duplication procedures vary by system and program. Planning should verify the applicable terms, installed hardware and authorized contacts rather than promise that a physical key can never be copied.

!Standard commercial keys may have been copied without clear records
!Key requests are not tied to an authorized contact
!Former employees or vendors may retain working keys

!Existing cylinders may not suit the proposed system
!Key issue and return records are inconsistent

Diagnosis and scope

Define the key-control problem

Before selecting a system, identify where uncontrolled copies, former key holders or unclear authorization create risk. The proposed controls should solve a documented operational problem.

Key points

  • Unknown or untracked key copies
  • Employee, tenant and vendor turnover
  • Different authorization levels by department
  • Unclear approval for replacement-key requests

Commercial decision support

Evaluate cylinders and changeover scope

Restricted-key planning may require cylinder changes when installed hardware is incompatible. Door function and lock condition should be separated from the key-control project so repair needs are scoped clearly.

Key points

  • Cylinder format and hardware inventory
  • Compatibility assessment by opening
  • Rekey, cylinder-change and hardware-repair separation
  • Phased changeover for occupied properties

Service details

Establish authorization and records

Technology alone does not manage keys. The property should identify who can request keys, how approval is documented, which access group each key belongs to and what happens during role changes.

Key points

  • Authorized requester list
  • Key issue and return records
  • Access groups by suite, team or function
  • Procedure for lost keys and staffing changes

Commercial FAQ

Questions about this commercial service

Does restricted key mean a key is impossible to copy?

No absolute duplication claim should be assumed. Controls depend on the selected system, its applicable terms and the authorization process used by the property.

Will a restricted-key project require new cylinders?

It may. Existing cylinders must be identified and checked for compatibility. The scope can distinguish reusable hardware from cylinders or locks that require a separate change.

Who should be authorized to request additional keys?

The business should name specific roles or contacts and define how requests are approved and recorded. Authorization should change when staff, tenants or responsibilities change.

What records support a commercial restricted-key program?

Maintain authorized requester details, each issued key and access group, issue and return dates, and the procedure for lost keys or staffing changes. The record format should match the property's approval workflow.
